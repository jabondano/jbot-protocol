# Marketing Performance Monitor
*Daily digest of ad performance, email campaigns, and automated insights*

---

## What This Is

A configurable system that pulls data from your ad platforms + email platform every day and delivers a structured performance brief — no dashboard login required.

**Used by a NASDAQ-listed company** to monitor $15-30K/month in ad spend across Meta, Google, and email with zero manual data pulls.

---

## The Problem

You're running paid ads + email campaigns. Every morning you ask:
- How much did we spend yesterday?
- What's our ROAS / ROI?
- Which campaigns are working? Which are burning budget?
- Any ads spending with zero conversions?
- Did we send any emails? How did they perform?

**Traditional approach:** Login to Meta Ads Manager, Google Ads, Klaviyo → pull data → calculate metrics → copy into Slack/spreadsheet → repeat tomorrow

**Time cost:** 15-30 minutes/day  
**Frequency:** Daily (or you miss trends)  
**Error rate:** High (manual calculations, copy-paste mistakes)  
**Insight quality:** Low (you're focused on pulling data, not analyzing it)

---

## The Solution

A **5-section automated daily brief** delivered to Telegram/Discord/Slack:

```
1. ACCOUNT SUMMARY (overall spend, revenue, ROAS, conversions)
2. TOP CAMPAIGNS (by spend, sorted high→low)
3. CREATIVE PERFORMANCE (winners/losers/alerts)
4. EMAIL PERFORMANCE (campaigns sent, timing)
5. INSIGHTS & RECOMMENDATIONS (what to do today)
```

**Every morning. No login. No manual pulls. No calculations.**

## How It Works

### The 5-Section Structure

#### Section 1: Account Summary
**What:** Overall performance across all campaigns  
**Metrics:** Spend, revenue, ROAS, CPA, CTR, CPC, impressions, clicks  
**Indicators:** 🟢 (above target), 🟡 (acceptable), 🔴 (below target)  
**Configured:** Thresholds per metric (e.g., ROAS ≥3x = 🟢)

#### Section 2: Campaign Performance
**What:** Top N campaigns by spend (default: 5)  
**Sorted:** Highest spend → lowest  
**Metrics:** Spend, ROAS, conversions per campaign  
**Why:** Identify where budget is going and if it's working

#### Section 3: Creative Analysis
**What:** Categorize individual ads into:
- **Winners:** ROAS ≥ threshold (default: 3x)
- **Losers:** ROAS < threshold + spend ≥ minimum (default: 2x, $10)
- **Alerts:** High spend ($30+) with zero conversions

**Sorted:** Winners by ROAS (high→low), Losers by ROAS (low→high), Alerts by spend (high→low)  
**Why:** Know what to scale, what to pause, what needs immediate attention

#### Section 4: Email Performance
**What:** Email campaigns sent in the last 24 hours  
**Data:** Campaign name, send time  
**Future:** Open rate, click rate, revenue per campaign (requires API access)

#### Section 5: Insights & Recommendations
**What:** Automated analysis based on thresholds  
**Examples:**
- "✅ Excellent ROAS - account performing above target"
- "🚨 URGENT: 3 ads burning $30+ with zero conversions - pause immediately"
- "🚀 12 winning ads (3x+ ROAS) - scale budget on top performers"
- "⚠️ Low CTR (<1%) - creative refresh recommended"

**Generated by:** Rule engine (if X metric crosses Y threshold, output Z insight)

## Configuration: Your Platforms, Your Thresholds

The structure is universal, but every company has different:

### What Gets Configured

**Ad Platforms:**
- Which platforms? (Meta, Google, TikTok, LinkedIn, Pinterest, etc.)
- Which metrics matter? (ROAS, CPA, CTR, CPC, etc.)
- API credentials

**Email Platform:**
- Which platform? (Klaviyo, Mailchimp, SendGrid, etc.)
- API credentials

**Thresholds:**
- What's a "good" ROAS? (3x, 2.5x, 4x — varies by industry/margin)
- What's a "high" CPA? ($40, $60, $100 — varies by product price)
- What's "high spend with zero conversions"? ($30, $50, $100 — varies by budget)
- What's "low CTR"? (1%, 0.5%, 2% — varies by channel/audience)

**Categories:**
- Winner threshold (default: ROAS ≥ 3x)
- Loser threshold (default: ROAS < 2x, spend ≥ $10)
- Alert threshold (default: spend ≥ $30, conversions = 0)

**Output:**
- Where to deliver? (Telegram, Discord, Slack, Email)
- Who gets it? (names for footer)
- Frequency? (daily, weekly, custom)

## Quick Start

### 1. Review Example Configurations

- **`examples/dtc-ecommerce-meta-klaviyo.json`** — Meta + Klaviyo, 3x ROAS target
- **`examples/b2b-saas-linkedin-hubspot.json`** — LinkedIn + HubSpot, $60 CPA target
- **`examples/multi-platform-agency.json`** — Meta + Google + TikTok, agency thresholds

### 2. Complete the Intake

Read `config/intake.md` and answer:
- Which ad platforms?
- Which email platform?
- What are your ROAS/CPA/CTR targets?
- Who gets the daily brief?

**Time:** ~20 minutes

### 3. Generate Configuration

```bash
node config/generate.js --intake your-responses.json
```

Outputs: `your-company.config.json`

### 4. Deploy as Cron Job

**If using OpenClaw:**
```bash
openclaw cron add \
  --name "daily-marketing-brief" \
  --schedule "0 9 * * *" \
  --agent-turn \
  --message "Run the marketing performance monitor with config: your-company.config.json"
```

**If standalone:**
```bash
# Add to crontab
0 9 * * * cd /path/to/scripts && python3 performance_monitor.py --config ../config/your-company.config.json
```

### 5. First Run (Manual)

```bash
python3 scripts/performance_monitor.py \
  --config config/your-company.config.json \
  --date yesterday \
  --output telegram
```

**Review:**
- Are thresholds set correctly? (🟢/🟡/🔴 indicators make sense?)
- Are categories useful? (winners/losers/alerts populated reasonably?)
- Is output format readable?

### 6. Iterate Thresholds

**Common adjustments:**
- ROAS target too high/low (adjust based on margin)
- CPA threshold off (adjust based on LTV)
- Alert spend threshold too sensitive (adjust based on daily budget)

**After 7-10 days:** Thresholds should stabilize based on real performance patterns

## Who This Is For

### ✅ Good Fit

**DTC e-commerce:**
- Running Meta/Google/TikTok ads
- Spend $500-50K/month
- Need daily visibility without logging in
- Team wants proactive alerts (not reactive dashboard checks)

**B2B SaaS:**
- Running LinkedIn/Google ads
- Longer sales cycles (weeks/months)
- Need to catch budget waste early
- Marketing team distributed (Slack/Telegram delivery preferred)

**Agencies:**
- Managing multiple client ad accounts
- Need client-specific thresholds
- Want daily check-ins without manual pulls
- Profitability depends on efficiency

### ❌ Not a Fit (Yet)

**Very early stage (<$500/month spend):**
- Volume too low for daily monitoring
- Manual spot-checks are fine
- Wait until spend justifies automation

**Enterprise with BI team:**
- Already have Looker/Tableau dashboards
- Data team builds custom reports
- This is redundant

**Offline/retail-only:**
- No digital ad spend
- Email-only campaigns (use platform dashboards)

## Comparison to Dashboards

| Feature | Dashboard | Daily Brief |
|---------|-----------|-------------|
| **Time to consume** | 10-15 min (login, navigate, pull data) | 2-3 min (scan brief) |
| **Insights** | You calculate | Pre-calculated + categorized |
| **Alerts** | You notice | Proactive (flagged in brief) |
| **Delivery** | Pull (you go get it) | Push (it comes to you) |
| **Team access** | Logins required | Shared channel (no login) |
| **Historical** | Yes (query historical) | No (point-in-time snapshot) |

**When to use dashboard:** Deep dives, historical analysis, detailed attribution  
**When to use daily brief:** Morning check-in, quick decision-making, proactive alerts

**They complement each other.** Brief for daily ops, dashboard for deep analysis.
